DOM

As a browser loads a website, it creates a model of the page. This model is called a DOM Tree.

  • Yellow: Document Node.
    Every element, attribute, text in HTML is represented by DOM unique node. It is usually at the top of a page doc, and represents the page itself.

  • Green: Element Node.
    HTML elements describe HTML document structure. To access DOM Tree, start looking for element nodes. After locating nodes, a nodes attributes/text nodes can be accessed.

  • Red: Attribute Node.
    Stuff like ID and classes. Attribute nodes are part of an element.

  • Purple: Text Node.
    Once a node is accessible, the text within can be reached and modified. They cannot have children.

Events: A browser's way of announcing that the user has done something. This allows for more interactivity in a website.

UI Events: Occurs when user interact with browser UI rather than webpage. (EG: load, unload, error, resize, scroll)

Keyboard Events: Occurs when user interacts with keyboard (aka inputs). (EG: keydown, keyup, keypress)

Mouse Events: Occurs when user interacts using mouse/touchpad/trackpad. (EG: click, dblclick, mousedown, mouseup, mousemove, mouseover, mouseout)

Focus Events: Occur when element gains/loses focus. (EG: focus/focusin, blur/focusout)

Form Events: Occurs when user interact with form elements. (EG: input, change, submit, reset, cut, copy, paste, select)
